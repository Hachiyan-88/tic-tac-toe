○×ゲーム（Tic-Tac-Toe）コンソールアプリ 仕様書
	
	（Java / オブジェクト指向化準備）
	
	1. 概要
	
	本アプリケーションは Java コンソールで動作する ○×ゲーム（Tic-Tac-Toe） である。
	3×3 の盤面で、プレイヤー（×）と CPU（〇）が交互に手を入力する。
	CPU は 2 段階の強さ（弱・強）を選択できる。
	
	この仕様書は、現状の実装（単一クラス構成）に基づきつつ、
	今後オブジェクト指向でクラス分割していくためのステップも含んでいる。
	
	2. 要件
	2.1 基本仕様
	
	盤面：3 × 3 の格子
	
	プレイヤーは「×」、CPU は「〇」
	
	縦・横・斜めのいずれか 3 つが揃えば勝利
	
	全マス埋まった場合は引き分け
	
	プレイヤーが行・列を入力し、空きマスに置く
	
	CPU の強さを起動時に選択できる
	
	1: 弱い（ランダム）
	
	2: 強い（ブロック & 勝ち手優先）
	
	2.2 CPU の行動
	弱い（cpuRandomMove）
	
	空いているマスをランダムに選択して置く。
	
	強い（cpuHardMove）
	
	以下の順で優先して打つ：
	
	CPU(〇) が勝てる手がないか探し、あれば置く
	
	プレイヤー(×) が次に勝てる場所をブロック
	
	上記がなければランダム
