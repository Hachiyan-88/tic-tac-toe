tictactoe_app

このリポジトリは、Java の学習の一環として作成した 〇×ゲーム（TicTacToe） です。
コンソール上で動作し、CPU の強さ選択・先攻後攻選択・勝敗判定・再戦機能など、
複数の Java の基礎要素を組み合わせて構成されています。

📁 プロジェクト概要

src/tictactoe/ 配下にゲーム全体を構成するクラス群がまとまっています。

🎮 機能一覧
1. CPU 対戦（プレイヤー vs CPU）

3×3 の盤面でプレイヤーと CPU が対戦

プレイヤーは行・列を入力してマスを選択

CPU のレベルは以下から選べる

1: 弱い（ランダム）

2: 強い（簡易アルゴリズム）

2. 先攻 / 後攻 の選択

ゲーム開始時にプレイヤーが先攻か後攻かを選択

TurnManager がターン管理および行動順を制御

Game クラスの複雑化を防ぐためにロジックを外部に分離

3. 勝敗判定 / 引き分け判定

GameController による一元管理

同一行 / 同一列 / 斜め一致をチェック

盤面が埋まっていれば自動的に引き分け

4. 再戦機能

ゲーム終了後、ReplayManager が y/n 入力を処理

nextLine() + parseInt を統一して使用するため、
入力バグ（nextInt 残りの改行問題）を解消済み

🧩 クラス構成
クラス名	役割
Game	メインループ、ゲームの流れ管理
Board	盤面の生成・表示・セルの管理
Player	プレイヤーの入力処理
CPU（抽象クラス）	CPU の共通処理
EasyCPU	弱い CPU（ランダム）
HardCPU	強い CPU（戦略あり）
GameController	勝敗判定、ターンの実行
TurnManager	先攻後攻とターン交代の管理
ReplayManager	y/n の再戦処理
InputUtil	nextLine + parseInt の安全入力処理
